1. 3706 神達 - 觀望；營收動能強，但 margin 還沒證明能完全轉成 EPS

公司白話介紹

神達做伺服器、工業電腦、車用與雲端資料中心相關硬體。白話說，它幫企業或資料中心做比較耐用、可長時間運轉的電腦系統，景氣會跟 AI/雲端、企業 IT 與伺服器需求連動。

1. 一句話結論

觀望；營收動能強，但 margin 還沒證明能完全轉成 EPS。現價隱含情境：Base 情境偏高。理由：3M 營收 YoY 約 41.5%、月營收 YoY 約 49.3%，P/E 仍在可研究區。 主要風險：Q1 營益率只有約 3.7%，若營收來自低毛利案型，股價題材會比獲利先跑。

2. 市場 regime 與投資組合閘門

  • 市場閘門：Normal。說明：本地篩選報告已更新加權指數至 2026-06-17，當天發行量加權股價指數 +0.1%，未觸發單日風險閘門。少數候選的日價歷史仍落後於價格日，操作上仍需依個股旗標確認資料完整性。
  • De-risk 白話：市場風險升高時，先降低追價與新買速度；不是所有股票都要賣，而是買點要更嚴格。
  • 投資組合：本輪是台股分析，只檢查台股可用現金、台股持股與同產業曝險；美股部位不納入此台股單股判斷。
  • 目前缺口：未取得台幣可用現金、現有台股持股與交易額度，因此不能計算可買股數。

3. 產業與國際情勢

  • 產業：電腦及週邊設備；白話類型：電腦週邊/伺服器與工業電腦。
  • 這類股票主要看：營收是否延續、毛利率/營益率是否撐住、股價是否已先反映太多好消息。
  • 國際變數：電子/半導體看全球科技景氣與客戶拉貨；航運看運價與大宗物料需求；化工看原料與產品報價；資訊服務看企業 IT 預算。
  • 結論：產業題材可以加分，但不能取代 EPS、現金流與估值位置。

4. Bear case first

  1. 營收高成長但毛利率/營益率偏低，可能是低毛利出貨放大。
  2. 估值溫度已偏熱，新增部位需要等回測或 EPS 上修。
  3. 若 AI/伺服器主題降溫，低 margin 標的容易先被 de-rating。

Bear case first 白話：先問「我會錯在哪裡」。它不是看空，而是避免只找支持買進的理由。若任一項發生：不加碼；若已持有同族群，先降風險或等正式月營收/季報確認。

11. 停損與停利

  • 若在估值進場區小量 starter：初始風控價約 58.6，並以基本面壞掉優先於價格。
  • 若只是技術試單：跌破 50D/技術試單區後，若接下來 2-3 個交易日站不回去，代表市場不願意接，停止新增風險。
  • 停利/降風險：>= 93.5 且沒有 EPS/margin 上修，分批停利或減碼檢查；高檔爆量長黑也先降風險。

Rerating Candidate；題材有，但必須看到 margin 轉換與現金流品質。 Rerating 白話是「市場願意給更高估值」；De-rating 是「市場把估值打回低一階」。若價格跌破 50D 或下次月營收/margin 不接上，用 Section 8 的 Stress/估值進場區重估；若 EPS/margin 明確上修，才重新抬高 overheat price。
